### 2. Instrument your application
**For Kubernetes deployments with zero-code instrumentation:**

```yaml
apiVersion: openlit.io/v1alpha1
kind: AutoInstrumentation
metadata:
  name: datadog-instrumentation
  namespace: default
spec:
  selector:
    matchLabels:
      instrument: "true"
  python:
    instrumentation:
      provider: "openlit"
      version: "latest"
  otlp:
    endpoint: "http://datadog-agent.default.svc.cluster.local:4318"
    timeout: 30
  resource:
    environment: "production"
    serviceName: "my-ai-service"
```

<Note>
Replace `datadog-agent.default.svc.cluster.local:4318` with your DataDog Agent service name and namespace. 
The DataDog Agent handles authentication and forwarding to DataDog.
</Note>

Refer to the [OpenLIT Operator Documentation](/latest/operator/overview) for more advanced configurations and use cases.
