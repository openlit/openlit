### 2. Instrument your application
**For Kubernetes deployments with zero-code instrumentation:**

```yaml
apiVersion: openlit.io/v1alpha1
kind: AutoInstrumentation
metadata:
  name: openobserve-instrumentation
  namespace: default
spec:
  selector:
    matchLabels:
      instrument: "true"
  python:
    instrumentation:
      provider: "openlit"
      version: "latest"
  otlp:
    endpoint: "YOUR_OPENOBSERVE_HTTP_ENDPOINT"
    headers: "YOUR_OPENOBSERVE_AUTH_HEADER"
    timeout: 30
  resource:
    environment: "production"
    serviceName: "my-ai-service"
```

Replace:
1. `YOUR_OPENOBSERVE_HTTP_ENDPOINT` with the Ingestion HTTP Endpoint value you copied in Step 1.
   * Example: `http://openobserve.monitoring.svc.cluster.local:5080/api/default` (for in-cluster)
   * Example: `http://127.0.0.1:5080/api/default` (for local development)
1. `YOUR_OPENOBSERVE_AUTH_HEADER` with the Ingestion Authorization key value you copied in Step 1.
   * Example: `Authorization=Basic%20<Ingestion Authorization key>`

Refer to the [OpenLIT Operator Documentation](/latest/operator/overview) for more advanced configurations and use cases.
