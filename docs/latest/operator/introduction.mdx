The **OpenLIT Operator** brings zero-code AI observability to Kubernetes environments, automatically instrumenting your AI applications with OpenTelemetry without requiring any code changes. Built specifically for AI workloads, it provides seamless observability for LLMs, vector databases, and AI frameworks running in Kubernetes.

## What is Zero-Code Instrumentation?

Zero-code instrumentation means your AI applications get full observability automatically:

- **No Code Changes**: Your existing applications work unchanged
- **Automatic Detection**: The operator detects AI libraries and frameworks
- **Seamless Integration**: Works with any OpenTelemetry-compatible backend
- **Production Ready**: Designed for enterprise Kubernetes environments

## üèóÔ∏è How It Works

<Steps>
  <Step title="Label Your Pods">
    Add labels to your AI application pods to indicate they should be instrumented
    ```yaml
    labels:
      openlit.io/instrument: "true"
    ```
  </Step>
  
  <Step title="Create AutoInstrumentation">
    Define instrumentation configuration using the AutoInstrumentation Custom Resource
    ```yaml
    apiVersion: openlit.io/v1alpha1
    kind: AutoInstrumentation
    spec:
      selector:
        matchLabels:
          openlit.io/instrument: "true"
      otlp:
        endpoint: "http://openlit:4318"
    ```
  </Step>
  
  <Step title="Automatic Injection">
    The operator automatically injects an init container that sets up OpenLIT instrumentation
  </Step>
  
  <Step title="Zero-Code Observability">
    Your applications automatically emit traces and metrics without any code changes
  </Step>
</Steps>
