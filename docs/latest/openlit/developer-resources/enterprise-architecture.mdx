# Enterprise Architecture Boundary

OpenLIT keeps the public repository as the OSS upstream. Shared contracts live in OSS so new features can start from a common shape, while paid implementations live in the private enterprise repository.

## OSS owns

- edition helpers such as `OPENLIT_EDITION`
- public feature metadata contracts at `src/client/src/features/index.ts`
- an empty OSS feature list in that normal feature module
- disabled-feature and upgrade-required response helpers
- OSS leak checks that prevent private implementation paths from landing here

## Enterprise owns

- paid feature implementations
- the populated EE feature list at `src/client/src/ee/features/index.ts`
- Enterprise-only implementation modules under `src/client/src/ee`
- license verification and application
- entitlement persistence and admin tools
- audit log storage, APIs, and pages
- cloud provisioning, internal admin APIs, and billing provider integrations

## Feature workflow

1. Add shared contracts or helper types in OSS only when the feature needs them.
2. Keep OSS `EDITION_FEATURES` empty unless the feature is available in CE.
3. Sync OSS into the enterprise repository.
4. Add the paid feature metadata to the enterprise `ee/features/index.ts`.
5. Implement the paid behavior under `src/client/src/ee`.
6. Gate paid routes with the enterprise feature-access layer.

OSS code may expose a stable contract for a paid feature, but it must not include the paid implementation or database schema for that feature.

Application code should import feature metadata from `@/features`. In OSS this resolves to the normal empty feature list. In Enterprise, path resolution prefers `src/ee/features` before `src/features`, so the same import resolves to the EE feature list.

Enterprise implementations should stay under `src/client/src/ee`, for example `src/client/src/ee/lib/audit`. Enterprise path aliases should resolve EE paths first and normal OSS paths second.

## Sync trigger

The OSS repository owns `.github/workflows/trigger-enterprise-sync.yml`. It runs on pushes to `main` and can also be manually triggered. The workflow dispatches `oss-main-updated` to `openlit/openlit-enterprise`, where the enterprise sync workflow creates a `sync/oss-main-YYYYMMDD` PR.

Configure `ENTERPRISE_SYNC_TOKEN` in the OSS repository secrets. The token must be able to dispatch events to `openlit/openlit-enterprise`.
