OpenGround lets you run the same prompt against multiple LLM providers and models simultaneously, then compare the responses, latency, token usage, and cost side-by-side. Every comparison run is saved, so you can build a searchable history of experiments and re-run any of them later with one click.

## Features

- **Multi-provider comparison**: Pick any combination of providers and models - built-in support for OpenAI, Anthropic, Cohere, Mistral, and Google, plus any custom/managed model you've added.
- **Custom and managed models**: Add your own model entries per provider (custom endpoint, custom pricing) and import/export pricing as an SDK-compatible `pricing.json`.
- **Prompt Hub integration**: Run a comparison using a free-text prompt, or pull a saved prompt (with a specific version and `{{variable}}` substitution) directly from [Prompt Hub](/latest/openlit/prompts-experiments/prompt-hub).
- **Per-provider run settings**: Configure temperature, max tokens, and other parameters independently for each provider in the comparison.
- **Results breakdown**: A metrics overview, a latency waterfall chart, and a cost breakdown are computed for every run so you can see which provider is fastest, cheapest, or most token-efficient at a glance.
- **Saved history with rerun**: Every run is persisted with its prompt, providers used, and best cost/speed/token-efficiency, listed on the main OpenGround page. Open any past run and click **Rerun** to reload it into a new comparison.
- **SDK usage snippets**: Once you've settled on a model, open the SDK usage dialog for ready-to-paste Python/Go snippets to instrument it directly with the OpenLIT SDK.

## Run a comparison

<Steps>
  <Step title="Start a new comparison">
    From the OpenGround history page, click **New** to open the comparison runner.
  </Step>
  <Step title="Choose your prompt source">
    Write a prompt directly, or switch to **Prompt Hub** and select a saved prompt, its version, and fill in any `{{variable}}` placeholders it defines.
  </Step>
  <Step title="Select providers and models">
    Use the provider grid to add one or more provider + model combinations to compare. Each selected combination gets its own settings panel for temperature, max tokens, and other run parameters.
  </Step>
  <Step title="Run and compare">
    Click **Compare**. Results stream in per provider as result cards, alongside a metrics overview, a latency waterfall, and a cost breakdown across all selected models.
  </Step>
  <Step title="Save and revisit">
    The run is saved automatically. Find it later on the OpenGround history page - sorted by prompt, providers used, and best cost/speed/token-efficiency - and click into it for the full detail view, or click **Rerun** to reload it as a new comparison.
  </Step>
</Steps>

## Custom and managed models

If a model you want to test isn't in the built-in provider list, add it from the model management dialog in OpenGround. You can set a custom endpoint and pricing for it, and it becomes available in the provider grid alongside the built-in models.

Custom model pricing can be exported as an SDK-compatible `pricing.json` and imported elsewhere - see [Manage Models](/latest/openlit/pricing/manage-models) for the full import/export workflow, since OpenGround and Manage Models share the same underlying provider/model store.

## Frequently asked questions

<AccordionGroup>
  <Accordion title="What is OpenGround used for?">
    OpenGround runs the same prompt across multiple LLM providers and models at once, so you can compare cost, latency, and response quality side-by-side before choosing a model for production.
  </Accordion>
  <Accordion title="Can I test models that aren't built in?">
    Yes. Add a custom or self-hosted model with its own endpoint and pricing via the model management dialog, and it appears in the provider grid alongside built-in models.
  </Accordion>
  <Accordion title="Are comparison runs saved automatically?">
    Yes. Every run is saved to a searchable history with cost/speed/token-efficiency summaries, and you can reopen any past run and click Rerun to reload it as a new comparison.
  </Accordion>
</AccordionGroup>
